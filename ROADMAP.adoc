== Overview

TypeLL is the unified type system verification kernel for the hyperpolymath ecosystem. It is to PanLL what LLVM is to Clang: the formal verification substrate that multiple language frontends compile _types_ into. TypeLL provides a 10-level type safety hierarchy used by VQL-UT, 007, Kategoria, PanLL, and the nextgen-languages suite.

== Current State (2026-03-29)

=== Architecture

```
Language Frontends (14 bridges)
        |
        v
typell-core (unified kernel)
  types.rs      — Unified type representation
  unify.rs      — Robinson unification + occurs check
  infer.rs      — Bidirectional type inference
  check.rs      — Type checking coordinator
  linear.rs     — Linear/affine usage tracking (QTT)
  effects.rs    — Algebraic effect system
  qtt.rs        — QTT semiring operations
  dimensional.rs — Dimensional analysis (SI units)
  session.rs    — Session type protocols
  proof.rs      — Proof obligation generation
  error.rs      — Diagnostic types
        |
        v
typell-server (HTTP/JSON-RPC on port 7800)
```

=== 10-Level Type Safety Hierarchy

[cols="1,2,2"]
|===
| Level | Name | TypeLL Concept

| L1  | Parse-time safety         | Well-formed AST
| L2  | Schema-binding safety     | Named type resolution
| L3  | Type-compatible operations | Unification + operator checking
| L4  | Null-safety               | Option types, totality checking
| L5  | Injection-proof safety    | Refinement predicates
| L6  | Result-type safety        | Return type inference
| L7  | Cardinality safety        | Bounded quantifiers
| L8  | Effect-tracking safety    | Algebraic effects
| L9  | Temporal safety           | Session types, state machines
| L10 | Linearity safety          | QTT bounded usage, linear types
|===

=== ABI/FFI Layer

* **Idris2 ABI:** `src/abi/Types.idr`, `Layout.idr`, `Foreign.idr` — dependent type proofs for interface correctness
* **Zig FFI:** `ffi/zig/src/main.zig` — C-compatible implementation with integration tests

=== Consumers

* **VQL-UT** — Type-safe query language (supersedes VQL-DT); levels 7-10 map VQL-DT legacy extensions
* **007** — Agent meta-language type system (tropical types, Five Facets)
* **Kategoria** — Category theory verification; Route alpha achieved L9 (session types via Brady indexed monad)
* **PanLL** — Cross-panel type intelligence (TypeLLService + TypeLLEngine in src/core/)
* **typed-wasm** — TypeLL 10-level type safety applied to WASM memory, multi-module
* **nextgen-languages** — 14 language frontends via bridge crates
